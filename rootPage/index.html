Get Started View on GitHub
qube-cli
$ qube run --image Voyrox:nodejs:25.2.0
# Pulling image...
# Creating container with isolated namespaces
# Container started in 0.12s
$ qube list
# CONTAINER ID IMAGE STATUS PORTS
# Qube-j9sn487 Voyrox:nodejs:25.2.0 Running 3000:3000
$ qube eval Qube-j9sn487 node --version
# v18.15.0
Why Qube?

Built for speed and simplicity

Qube provides a qube-like workflow with a fraction of the overhead, perfect for development environments and CI/CD pipelines.

Lightning Fast

Written in Go for minimal overhead and sub-second container startup times. Perfect for rapid development iterations.

Secure Isolation

Leverages Linux namespaces and cgroups v2 for strong isolation without the complexity of traditional container runtimes.

Start Secure, Stay Secure

Security & Simplicity, By Default

Qube images and builds are designed for minimal attack surface, rapid CVE response, and enterprise-grade provenance.

Up to 97% reduction in attack surface

Shrink your image footprint and attack risk with ultra-minimal, Debian- and Alpine-compatible builds, with a one line update to your qubefile.

Up to 99% CVE reduction

Eliminate vulnerabilities before they reach production with continuously rebuilt images from Qube’s hardened pipeline and verified SBOMs.

7-day SLA for critical CVE remediation

Every image is built from trusted sources meeting SLSA Build Level 3 specifications with full SBOMs and verifiable provenance.

1000+ images and applications

Continuously growing catalog of FIPS and STIG ready languages, frameworks, databases, application images and Helm charts, all signed and verified with SLA-backed security.
